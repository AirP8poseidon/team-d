# Before / After — 수작업 → 에이전트화

## 우리가 자동화한 반복 수작업
사내 HPC 사용자가 **서버 환경·사용법·노하우를 매번 구두/메신저로 묻고, "지금 누가 어느 노드를 쓰는지"를 반복 확인**하던 수작업을, 검색되는 위키 + 실시간 채팅 위젯이 있는 웹 포털로 집약. (내 파트 = 정보공유 위키 + 채팅 위젯)

## Before (기존 수작업 절차)
1. 신규/타팀 사용자가 서버 스펙·접속법·모듈 구성을 **선임에게 매번 구두 문의**.
2. 장애·OOM·Slurm PENDING 같은 노하우는 **개인 메모·메신저에 산재** → 검색 불가, 시간이 지나면 휘발.
3. 노드 충돌을 막으려 **"지금 node04 누가 써요?"를 메신저에 반복 질문**, 답을 기다림.
4. 같은 질문/답변이 사람만 바뀌며 무한 반복.

## After (에이전트화된 절차)
1. 위키 1페이지에서 **서버 3대(gpu/cpu/fat) 환경 + ssh→module→sbatch 가이드**를 즉시 열람.
2. 노하우는 **게시판에 누적·제목/본문 검색**(`/api/wiki/posts?q=`), 작성 즉시 반영.
3. 어느 페이지서든 뜨는 **플로팅 채팅 위젯**으로 "node04 18시까지 학습" 같은 사용 계획을 실시간 공유(2~3초 폴링).
4. FastAPI+SQLite로 기록이 남아 **재방문·검색 가능**, 신규자는 자가 학습.

## 효과 (형식 자유)

| 지표(내 파트 기준) | Before | After | 효과 |
|------|--------|-------|------|
| 서버 환경/사용법 확인 | 구두 문의·흩어진 메모 | 위키 1페이지 집약 | 탐색→즉시 열람, 반복 문의 제거 |
| 노하우 검색 | 불가(메신저 산재) | 제목·본문 q검색 | 8건 시드 + 작성 즉시 검색 |
| 노드 사용 소통 | 메신저 반복 질문·대기 | 실시간 채팅 위젯 폴링 | 비동기 질문→실시간 공유 |
| 채팅 위젯 재사용 | 없음 | `<script>` 한 줄 주입 | 4개 페이지 동일 위젯 0추가비용 |
| 동작 검증 | 수동 클릭 | TestClient 8종 자동 + 브라우저 MCP | 회귀 즉시 확인 |

> `PROCESS_LOG.md`의 효과 측정과 같은 지표.
