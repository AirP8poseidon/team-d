# 핵심 프롬프트 모음 — 실제로 입력한 지시 그대로

> 다른 팀이 그대로 따라 할 수 있도록 **실제 프롬프트**를 단계별로 남긴다. 상세 맥락은 `../PROCESS_LOG.md` 각 항목 참조.

## 1) 설계 — 인터뷰로 청사진 1문서 수렴 (#1~#5)
- 모호한 아이디어를 바로 코딩하지 말고 **인터뷰 → 확정 스펙 1문서(DEV_BLUEPRINT) → 팀원별 핸드오버** 로 분해.
  > "역할·페이지·소유 파일·DB·API·통합 접점을 한 문서로 확정해줘. 세 사람이 같은 파일을 절대 동시에 안 만지게 경계를 설계."
  > "팀원1·2가 받아서 바로 시작할 수 있게 자기완결 핸드오버 + 붙여넣는 킥오프 프롬프트를 만들어줘."
- **효과**: 새 세션·새 팀원도 문서 하나로 착수, 파일 충돌 0 설계.

## 2) 엔지니어링 잠금 + 뼈대 스캐폴딩 (#10~#12)
  > "/plan-eng-review 로 아키텍처를 잠가줘(동시성·멱등시드·파서가드 등)."
  > "공통 뼈대(빈 5페이지 + 라우터 5개 등록 + 수집 seam)를 먼저 만들어 병렬 착수 신호를 줘."
- 수집 seam(Collector ABC) = mock/실연동을 drop-in 교체. **main·라우터 변경 없이** 데이터 소스만 바꿈.

## 3) 실서버 직접 연동 (#17~#18) — 부하 최소
  > "여기서는 서버와 직접 통신이 가능하니 ... 마스터에서만 각 노드 정보 수집하고, 마스터랑 여기랑만 통신."
  > "원격서버 부하가 최소한으로 걸리도록 주의해서 작업할것."
- **패턴**: 포털PC→master SSH→노드 순회(`scan_once.sh`, 읽기전용·순차·1회성) → scp → ingest. 루프/상주 에이전트 없음.

## 4) CPU 중심 reframe + 팀원 통합 (#19~#22)
  > "모니터링 UI를 CPU 중심으로 reframe — GPU 막대/차트 → CPU%·MEM·현재 모델작업(사용자), Job 테이블이 주인공."
  > "팀원들의 작업이 각 이름 브랜치로 올라왔음. 이거 풀해서 완성본 만들것."
  > "남은 다듬기 해줘" (노드명 전 페이지 실노드 기준 통일)
- **통합 원칙**: 소유 파일 분리 → `git pull` 충돌 0. 개인 로그는 이름별 파일로 분리 보존(전원참여).

## 5) 실데이터 충실도 + 운영 (#23~#24)
  > "온도 잡기 시도해보고 안되면 목업 더미로 만들어서 가시화만 해도 좋음."  → 실측(NIC sensors) 우선, 없으면 CPU부하 추정.
  > "서버 올려주고 외부에서 접근 가능하게 웹서버 열어줘" / "저빈도 갱신 걸어주고 ... 인덱스 GPU→CPU."
- **운영**: `run_server.bat`(0.0.0.0:8000·ingest) + `refresh_loop.ps1`(저빈도 라이브 갱신) + 방화벽 1줄(LAN 한정).

## 재사용 팁 (왜 효과적이었나)
- **인터뷰→1문서→파생**: 의도를 먼저 문서로 수렴하면 팀원·새 세션이 흔들림 없이 병렬 착수.
- **수집 seam 분리**: mock 으로 개발 → 실데이터는 컬렉터만 교체. 데모 안전(폴백) + 실연동 동시 만족.
- **소유 파일 경계 명문화**: 3인 병렬 → 통합이 "pull + 접점 점검"으로 끝남.
- **부하·보안 가드레일을 프롬프트에 명시**: "읽기전용·1회성·LAN 한정" 같은 제약을 처음부터 박아두면 안전한 산출물.
